hooks -- list hooks attached to a key

Synopsis

  • Usage:
    hooks key
    hooks store
    hooks(store, key)
  • Inputs:
    • key, a sequence or a symbol,
    • store, a hash table, the hash table where the hooks are stored
  • Optional inputs:
    • Strategy => a thing, default value null, only list hooks with the given strategy
  • Outputs:
    • a numbered vertical list, of those hooks associated with key or stored in store

Description

i1 : hooks(intersect, Ideal, Ideal)

o1 = {0 => (intersect, Ideal, Ideal, Strategy => Default)    }
     {1 => (intersect, Ideal, Ideal, Strategy => Elimination)}
     {2 => (intersect, Ideal, Ideal, Strategy => Monomial)   }

o1 : NumberedVerticalList

See also

Ways to use hooks :

  • "hooks(HashTable)"
  • "hooks(List)"
  • "hooks(Sequence)"
  • "hooks(Symbol)"
  • "hooks(Thing)"
  • "hooks(ZZ)"

For the programmer

The object hooks is a method function with a single argument.
