Macaulay2Doc :: File << Thing

File << Thing -- print to a file

Synopsis

  • Operator: <<
  • Usage:
    f << x
    << x
  • Inputs:
    • f, a file, a string, or null
    • x, a thing
  • Outputs:
    • the output file(s) used, a file
  • Consequences:
    • The object x is prepared for printing (with net) and printed on the output file(s) f. If f is a string, then it is interpreted as a filename and an output file is opened, used, and returned, unless a single open file with the same name already exists, in which case it is used and returned. Filenames starting with ! or with $ are treated specially, see openInOut. If f is a list, then the output operation is performed on each one. If f is null, then the output is discarded; thus null is useful as a dummy output file. If f is omitted, as in the second usage line, then the output is sent to stdio, and it will appear (usually) on the screen.

Description

Parsing of << associates leftward, so that several objects may be displayed with an expression such as f<<x<<y<<z.

i1 : stderr << "-- hi there --" << endl
-- hi there --

o1 = stderr

o1 : File
 
i2 : << "-- ho there --" << endl
-- ho there --

o2 = stdio

o2 : File
 
i3 : fn = temporaryFileName()

o3 = /private/tmp/M2-64106-0/0
 
i4 : fn << "hi there" << endl << close

o4 = /private/tmp/M2-64106-0/0

o4 : File
 
i5 : get fn

o5 = hi there
