NCAlgebra :: resolution(NCMatrix)

resolution(NCMatrix) -- Compute the resolution of coker M as a map of free right modules

Synopsis

  • Function: resolution
  • Usage:
    res M
  • Inputs:
    • M, an instance of the type NCMatrix,
  • Optional inputs:
    • LengthLimit => an integer, default value infinity, The length of the resolution. This defaults to the number of variables of ring M.
    • DegreeLimit => ..., default value null,
    • FastNonminimal => ..., default value false,
    • HardDegreeLimit => ..., default value {},
    • PairLimit => ..., default value infinity,
    • SortStrategy => ..., default value 0,
    • StopBeforeComputation => ..., default value false,
    • Strategy => ..., default value null,
    • SyzygyLimit => ..., default value infinity,
  • Outputs:
    • an instance of the type NCChainComplex,

Description

This function computes a minimal graded free resolution of the cokernel of M, considered as a map of graded right free modules. M must be homogeneous for this command to work.

As of this version, NCChainComplex (the return type) is still quite simple, though betti still works on them.

i1 : A = skewPolynomialRing(QQ,(-1)_QQ,{x,y,z})
--Calling Bergman for NCGB calculation.
Complete!

o1 = A

o1 : NCQuotientRing
 
i2 : M = ncMatrix {{x,y,z}}

o2 = | x y z |

o2 : NCMatrix
