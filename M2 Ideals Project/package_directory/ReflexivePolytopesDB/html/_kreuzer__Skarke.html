ReflexivePolytopesDB :: kreuzerSkarke

kreuzerSkarke -- access Kreuzer-Skarke dimension 4 reflexive polytopes database

Synopsis

  • Usage:
    kreuzerSkarke(h11, Limit=>n)
    kreuzerSkarke(h11, h12)
    kreuzerSkarke str
  • Inputs:
    • h11, an integer, The desired Picard number of the Calabi-Yau hypersurface, a number between 1 and 491 inclusive.
    • h12, an integer, (optional) the desired $h^{1,2}$ of the Calabi-Yau hypersurface
    • str, a string, parses the given string into a list of KSEntry's
  • Optional inputs:
    • Limit (missing documentation) => an integer, default value 1000, The maximum number of examples to retrieve
    • Vertices (missing documentation) => an integer, default value null, Restrict to those examples with this number of vertices
    • Facets (missing documentation) => an integer, default value null, Restrict to those examples with this number of facets
    • LatticePoints (missing documentation) => an integer, default value null, Restrict to those examples with this number of lattice points
    • DualLatticePoints (missing documentation) => an integer, default value null, Restrict to those examples whose polar dual has this number of lattice points
    • H12 (missing documentation) => an integer, default value null, Restrict to those examples whose associated Calabi-Yau 3-fold has the given $h^{1,2}(X)$
    • Access (missing documentation) => a string, default value null, Either null (use Macaulay2's getWWW), "wget" or "curl". Mainly an internal optional argument for developers
  • Outputs:
    • a list, of KSEntry's, each of which is one entry (one polytope) from the database. Use matrix(KSEntry) and description(KSEntry) to extract information from an entry.

Caveat

The database is organized by the h11 value. Placing filters can significantly slow down the database access. Using any of the optional arguments other than Limit also means that the locally stored datafiles will not be used.
